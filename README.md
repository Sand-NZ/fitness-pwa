# 🏋️ 健身实况记录 — Fitness Logger PWA

> 纯前端 · 无后端 · 全部数据存储在本地 · 离线可用

一款完全离线的渐进式 Web 应用（PWA），用于记录健身训练。支持自由训练与计划模式，动作字段可自定义，所有数据存储在浏览器本地。

**在线体验：** https://sand-nz.github.io/fitness-pwa/

---

## ✨ 功能

| 功能 | 说明 |
|---|---|
| **🏗️ 预置动作库** | 内置 37 个动作，分热身/推/拉/腿/核心 5 类，开箱即用 |
| **📋 预置计划** | 推日、拉日、腿日 3 个计划，可直接开始训练 |
| **➕ 自定义动作** | 动作字段自定义（重量/次数/时长/选择等），完全灵活 |
| **🏷️ 层级标签系统** | 标签支持父子层级、颜色和渐变（需自行创建） |
| **🏋️ 自由训练** | 选动作即练，可随时换动作 |
| **📋 计划模式** | 按计划顺序执行动作，组数自动积累 |
| **⏱️ 组间计时器** | 倒计时/秒表，结束时发声+振动 |
| **📝 训练记录** | 自动保存所有组数据，统计页可查看详情或删除 |
| **📊 统计面板** | 训练次数、总组数、总容量、平均 RPE、趋势图、CSV 导出 |
| **🌙 深色模式** | 跟随系统/浅色/深色/护眼琥珀，支持日出日落自动切换 |
| **💾 数据备份** | 全量/增量导出 JSON，导入支持合并或覆盖模式 |
| **🔒 加密导出** | 可设密码，使用 Web Crypto API (AES-GCM) 加密 |
| **📱 PWA 安装** | 添加到主屏幕后全屏运行，离线可用 |

## 🚀 快速开始

### 方式一：安装使用（推荐）

1. 用 Chrome 打开 https://sand-nz.github.io/fitness-pwa/
2. 地址栏出现安装图标 → 点击添加到主屏幕
3. 安装后像原生 App 一样全屏使用，**离线完全可用**

### 方式二：本地运行

```bash
# 用任意 HTTP 服务器启动
npx serve fitness-pwa

# 或用 Python
cd fitness-pwa && python3 -m http.server 8080
```

浏览器打开 `http://localhost:8080`。

## 🧩 预置数据

首次打开时自动载入 **32 个动作**（按名称去重，不覆盖自建）和 **3 个预置计划**：

- **热身** (7)：猫牛式、单杠拉背、蜘蛛趴、弹力带训练、呼吸训练、伟大者拉伸、俯身臂屈伸
- **推** (8)：杠铃卧推、哑铃卧推、双杠臂屈伸（助力）、上斜哑铃卧推、反向飞鸟、蝴蝶机夹胸、器械直臂下压、俯身哑铃后束飞鸟
- **拉** (8)：助力引体向上、高位下拉、T杆划船、钢索划船、单臂哑铃划船、面拉、哑铃推肩、静态悬挂
- **腿** (5)：保加利亚蹲、罗马尼亚硬拉、后腿弓步蹲、山羊挺身、哥本哈根支撑
- **核心** (4)：平板支撑、锯式俯卧撑、卷腹、仰卧抬腿

**预置计划：** 推日、拉日、腿日

## 🧩 数据模型

所有数据使用 **localStorage** 持久化，共 5 个集合：

| 集合 | 说明 |
|---|---|
| `tags` | 标签（层级、颜色、渐变） |
| `exercises` | 动作（自定义字段数组） |
| `plans` | 训练计划（引用动作 + 字段覆盖） |
| `records` | 训练记录（体重、RPE、各组数据） |
| `settings` | 设置（主题、模式、密码等） |

详细的模型结构见 [`js/models.js`](js/models.js)。

---

## 🔄 版本与迁移

版本号记录在 `STORAGE.currentVersion`。当数据模型发生变更时：

1. 递增版本号
2. 用 `STORAGE.registerMigration(version, fn)` 注册迁移函数
3. 下次启动自动执行，用户无感

参见 [`js/storage.js`](js/storage.js) 中的迁移示例。
